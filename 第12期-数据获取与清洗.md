## 📌 一句话概括
AKShare是Python金融数据获取与清洗的核心工具，为量化交易提供标准化数据接口，是量化策略开发的基础设施。

## 📖 概念详解

### 什么是AKShare？
AKShare是一个开源的Python金融数据接口库，专门用于获取和清洗金融数据。它提供了标准化的API接口，涵盖股票、期货、基金、外汇等多个金融市场的数据。通过AKShare，量化交易者可以轻松获取实时行情、历史K线、财务指标、资金流向等全维度数据，为策略开发和回测提供数据支持。

### 数据获取的重要性
在量化交易中，数据质量直接决定策略表现。高质量的数据能够确保策略的可靠性和稳定性，而脏数据则可能导致策略失效甚至产生错误的交易信号。数据获取不仅仅是技术问题，更是量化交易成功的基础。没有干净、准确的数据，再复杂的策略模型也难以产生持续稳定的收益。

### 数据清洗的关键步骤
数据清洗是将原始金融数据转化为可用分析数据的过程，主要包括：处理缺失值（如停牌日数据）、异常值检测（如极端价格波动）、数据格式统一（如时间戳格式标准化）、数据对齐（如不同数据源的时间同步）等。这些步骤确保了数据的完整性和一致性，为后续的策略开发和回测奠定坚实基础。

## 📐 数学公式

### 数据质量评估公式
```
数据质量分数 = (1 - 缺失值比例) × 0.3 + 
               (1 - 异常值比例) × 0.4 + 
               数据完整性得分 × 0.3
```

### 缺失值处理公式
```
均值填充：x̄ = (1/n) × Σxi
前向填充：xt = xt-1 (当xt为空时)
线性插值：xt = xt-1 + (xt+1 - xt-1) × (t - t-1) / (t+1 - t-1)
```

### 异常值检测公式（Z-Score方法）
```
Z-Score = (xi - μ) / σ
其中：
- xi: 单个数据点
- μ: 数据均值
- σ: 数据标准差
当 |Z-Score| > 3 时，认为该数据点为异常值
```

## 🐍 Python 代码实战

```python
import akshare as ak
import pandas as pd
import numpy as np
from datetime import datetime, timedelta

class AKShareDataProcessor:
    """AKShare数据获取与清洗处理器"""
    
    def __init__(self):
        self.stock_data = None
        self.processed_data = None
        
    def fetch_stock_data(self, stock_code="000001", start_date="2023-01-01", end_date="2023-12-31"):
        """获取股票历史数据"""
        try:
            # 使用AKShare获取股票数据
            self.stock_data = ak.stock_zh_a_hist(
                symbol=stock_code,
                period="daily",
                start_date=start_date,
                end_date=end_date,
                adjust="qfq"  # 前复权
            )
            print(f"成功获取 {len(self.stock_data)} 条数据")
            return self.stock_data
        except Exception as e:
            print(f"数据获取失败: {e}")
            return None
    
    def clean_data(self):
        """数据清洗流程"""
        if self.stock_data is None:
            print("请先获取数据")
            return None
            
        # 复制数据
        cleaned_data = self.stock_data.copy()
        
        # 1. 处理缺失值
        print("处理缺失值...")
        missing_values = cleaned_data.isnull().sum()
        print(f"缺失值统计:\n{missing_values}")
        
        # 对数值型列进行前向填充
        numeric_columns = cleaned_data.select_dtypes(include=[np.number]).columns
        cleaned_data[numeric_columns] = cleaned_data[numeric_columns].fillna(method='ffill')
        
        # 2. 处理异常值
        print("检测异常值...")
        for column in numeric_columns:
            if column in ['收盘', '开盘', '最高', '最低']:
                # 使用Z-Score方法检测异常值
                z_scores = np.abs((cleaned_data[column] - cleaned_data[column].mean()) / cleaned_data[column].std())
                outliers = cleaned_data[z_scores > 3]
                print(f"{column} 列异常值数量: {len(outliers)}")
                
                # 用中位数替换异常值
                median_value = cleaned_data[column].median()
                cleaned_data.loc[z_scores > 3, column] = median_value
        
        # 3. 数据格式标准化
        print("标准化数据格式...")
        # 确保日期格式正确
        if '日期' in cleaned_data.columns:
            cleaned_data['日期'] = pd.to_datetime(cleaned_data['日期'])
        
        # 确保数值型数据格式正确
        for column in numeric_columns:
            cleaned_data[column] = pd.to_numeric(cleaned_data[column], errors='coerce')
        
        # 4. 计算技术指标
        print("计算技术指标...")
        cleaned_data['涨跌幅'] = (cleaned_data['收盘'] - cleaned_data['开盘']) / cleaned_data['开盘'] * 100
        cleaned_data['振幅'] = (cleaned_data['最高'] - cleaned_data['最低']) / cleaned_data['开盘'] * 100
        cleaned_data['成交量变化'] = cleaned_data['成交量'].pct_change()
        
        # 5. 数据排序
        cleaned_data = cleaned_data.sort_values('日期').reset_index(drop=True)
        
        self.processed_data = cleaned_data
        print("数据清洗完成！")
        return cleaned_data
    
    def data_quality_check(self):
        """数据质量检查"""
        if self.processed_data is None:
            print("请先清洗数据")
            return None
            
        quality_report = {
            '总记录数': len(self.processed_data),
            '缺失值总数': self.processed_data.isnull().sum().sum(),
            '重复记录数': self.processed_data.duplicated().sum(),
            '日期范围': f"{self.processed_data['日期'].min()} 到 {self.processed_data['日期'].max()}",
            '数据完整性': (1 - self.processed_data.isnull().sum().sum() / (len(self.processed_data) * len(self.processed_data.columns))) * 100
        }
        
        print("=== 数据质量报告 ===")
        for key, value in quality_report.items():
            print(f"{key}: {value}")
        
        return quality_report
    
    def save_processed_data(self, filename="processed_stock_data.csv"):
        """保存处理后的数据"""
        if self.processed_data is not None:
            self.processed_data.to_csv(filename, index=False, encoding='utf-8-sig')
            print(f"数据已保存到 {filename}")
        else:
            print("没有数据可保存")

# 使用示例
if __name__ == "__main__":
    # 创建处理器实例
    processor = AKShareDataProcessor()
    
    # 获取数据（以平安银行为例）
    print("=== 获取股票数据 ===")
    processor.fetch_stock_data("000001", "2023-01-01", "2023-12-31")
    
    # 数据清洗
    print("\n=== 数据清洗 ===")
    cleaned_data = processor.clean_data()
    
    # 数据质量检查
    print("\n=== 数据质量检查 ===")
    quality_report = processor.data_quality_check()
    
    # 保存数据
    print("\n=== 保存数据 ===")
    processor.save_processed_data()
    
    # 显示前5行数据
    print("\n=== 清洗后的数据预览 ===")
    print(cleaned_data.head())
```

def fetch_multiple_stocks(stock_list, start_date, end_date):
    """批量获取多只股票数据"""
    all_data = []
    
    for stock_code in stock_list:
        try:
            data = ak.stock_zh_a_hist(
                symbol=stock_code,
                period="daily",
                start_date=start_date,
                end_date=end_date,
                adjust="qfq"
            )
            data['股票代码'] = stock_code
            all_data.append(data)
            print(f"成功获取 {stock_code} 数据")
        except Exception as e:
            print(f"获取 {stock_code} 数据失败: {e}")
    
    # 合并所有数据
    combined_data = pd.concat(all_data, ignore_index=True)
    return combined_data

## ⚠️ 常见陷阱

### 1. 数据时区问题
- **问题**：不同数据源可能使用不同的时区（UTC、北京时间等），导致时间对齐错误
- **解决方案**：统一使用北京时间（Asia/Shanghai），在获取数据后立即转换时区
```python
# 时区转换示例
cleaned_data['日期'] = pd.to_datetime(cleaned_data['日期']).dt.tz_localize('UTC').dt.tz_convert('Asia/Shanghai')
```

### 2. 缺失值处理不当
- **问题**：简单删除包含缺失值的记录，可能导致数据量不足
- **解决方案**：根据缺失原因选择合适的填充方法
```python
# 根据数据类型选择填充方法
def handle_missing_values(data):
    # 数值型数据用前向填充
    numeric_data = data.select_dtypes(include=[np.number])
    data[numeric_data.columns] = numeric_data.fillna(method='ffill')
    
    # 分类数据用众数填充
    categorical_data = data.select_dtypes(include=['object'])
    for col in categorical_data.columns:
        data[col] = data[col].fillna(data[col].mode()[0])
    
    return data
```

### 3. 异常值误判
- **问题**：将正常的市场波动误判为异常值，丢失重要信息
- **解决方案**：结合市场情况和业务逻辑判断异常值
```python
# 改进的异常值检测
def detect_outliers_with_context(data, column, threshold=3):
    """结合市场情况的异常值检测"""
    z_scores = np.abs((data[column] - data[column].mean()) / data[column].std())
    
    # 排除停牌日（成交量为0）
    trading_days = data['成交量'] > 0
    
    # 只在交易日检测异常值
    outliers = data[z_scores > threshold & trading_days]
    return outliers
```

### 4. 数据重复问题
- **问题**：同一时间点的数据被重复记录
- **解决方案**：基于时间戳和股票代码去重
```python
# 去重处理
def remove_duplicates(data, time_column='日期', stock_column='股票代码'):
    """基于时间戳和股票代码去重"""
    data = data.drop_duplicates(subset=[time_column, stock_column], keep='first')
    return data.reset_index(drop=True)
```
